# Step 1: Specify + CLAUDE.md 作成（5分）— 仕様を書き、AIに仕様の存在を教える

## やること（前半 3分）: SPEC.md 記入
`SPEC.md` を開いて、穴埋め部分をチームで議論しながら記入してください。

### 特に議論すべきポイント
- **1人何票まで？** → チームで議論して決めてください
- **投票の取り消しは？** → チームで議論して決めてください
- **ボタン連打対策は？** → チームで判断

### コツ
- 完璧でなくてOK。「書いてあるか、書いてないか」が重要
- 曖昧なまま放置しない。「決めきれないもの」は明示的に「TBD」と書く

---

## やること（後半 2分）: CLAUDE.md 作成

Claude Code は **CLAUDE.md を自動で読みます**。
SPEC.md を書いただけでは、AIはその存在を知りません。

- **CLAUDE.md なし** → 毎回プロンプトで「SPEC.md読んで」と言う必要がある
- **CLAUDE.md あり** → AIが自動的にSPEC.mdを参照する

CLAUDE.md を開いて、`<!-- Phase 3 でここに... -->` のコメント行を **削除** してから、以下を記入:

```
# プロジェクト指示

## SSoT（Single Source of Truth）
このプロジェクトの仕様書は SPEC.md です。
実装・修正・判断はすべて SPEC.md を参照してください。
SPEC.md に記載のない機能は実装しないでください。
仕様変更が入った場合は、まず SPEC.md を更新してからコードを修正してください。
```

> 参考: `templates/CLAUDE-template.md` にテンプレートがあります
> Phase 4 でこの CLAUDE.md をさらに育てます！
