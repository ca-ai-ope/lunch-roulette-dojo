# Story 002: 投票機能

## Why（なぜ作るか）
チームメンバーが「行きたい」お店に投票して、事前に候補を絞り込めるようにしたい。

## What（機能要件）

### 必須要件
- [ ] 各レストランカードに投票ボタンを追加する
- [ ] 投票数をカード上にリアルタイムで表示する
- [ ] 1ユーザーにつき、1つのレストランに1票まで
- [ ] 投票の取り消しが可能
- [ ] 投票状態がリロードしても保持される

### 受け入れ条件
1. 未投票のレストランには「投票する」ボタンが表示される
2. 投票済みのレストランには「投票済み」（取り消しボタン付き）が表示される
3. 投票ボタンを押すと即座に投票数が +1 される
4. 取り消しボタンを押すと即座に投票数が -1 される
5. 同じレストランに2回投票はできない
6. ブラウザをリロードしても投票状態が維持される
7. 既存のルーレット機能が正常に動作すること

## How（技術制約）
- 既存の `RestaurantCard.tsx` に投票UIを統合する
- 投票データは localStorage に保存する（`lib/storage.ts` を拡張）
- ユーザー識別は localStorage に保存した visitorId で行う
- 新規コンポーネント `VoteButton.tsx` を作成
- 投票ロジックのカスタムフック `hooks/useVote.ts` を作成
- `lib/types.ts` に VoteRecord 型を追加

## Edge Cases
- 同じユーザーが同じレストランに連打した場合 → 1票のまま
- localStorage が無効化されている場合 → エラーメッセージを表示
- 投票数が0未満にならないこと
- visitorId が未生成の場合 → 初回アクセス時に自動生成
