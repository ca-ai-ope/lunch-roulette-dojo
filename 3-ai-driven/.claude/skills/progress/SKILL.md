プロジェクトの進捗状況を確認します。

このスキルはファイルを変更しない。いつ実行しても安全。

## Step 1: 全ストーリーのステータスを表示する

stories/ 配下の全ディレクトリの CONTEXT.json を読み、以下の形式で表示する:

```
## プロジェクト進捗

| # | ストーリー | ステータス |
|---|---|---|
| story-001 | ランチルーレット基本機能 | ✅ completed |
| story-002 | 投票機能 | 🔧 in_progress |
```

ステータスのアイコン:
- `completed` → ✅
- `in_progress` → 🔧
- `pending` → ⏳

## Step 2: 進行中/未着手ストーリーの受け入れ条件を表示する

status が `in_progress` または `pending` のストーリーについて、
CONTEXT.json の `acceptance_criteria` を以下の形式で表示する:

```
## story-002: 投票機能 の受け入れ条件

✅ 投票ボタンが表示される
✅ 投票済み状態が表示される
✅ 投票数が即座に反映される
□ 取り消しができる
□ 同じお店に2回投票できない
□ リロードしても投票が保持される
□ ルーレット機能が壊れていない

達成: 3 / 7（43%）
```

- `true` → ✅
- `false` → □
- 達成率はパーセント付きで表示する

## Step 3: 次のアクションを提案する

状態に応じて**具体的な**次のアクションを1行で提案する:

- 全て pending → 「`/start` で story-XXX の実装を開始してください」
- in_progress で未達成あり → 「残り N 件: □ の条件を実装してください」
- in_progress で全達成 → 「全条件達成。`/finish` で完了記録を行ってください」
- 全 completed → 「全ストーリー完了。`/story` で新しいストーリーを作成できます」
