# Phase 2: Vibe Codingの限界 — 投票機能を追加

## メインプロンプト（これをClaude Codeにそのままコピペ）

```
チームで事前に候補を絞りたいから、投票機能つけて
```

## 別パターン

```
各お店に「いいね」ボタンつけて。人気順で並べたい。
```

## ルール
- **Claude Code は閉じない** — Phase 1 と同じセッションで続ける
- Phase 1 のアプリに **そのまま追加** する
- **仕様書は書かない**（Vibe Codingを継続）
- 壊れてもOK — むしろ壊れることが学び

## ⏱ 制限時間: 10分

## 起きるかもしれないこと
| 症状 | 具体例 |
|---|---|
| 状態管理破綻 | 投票数とUIが不一致 |
| 既存機能崩壊 | ルーレットが動かなくなる |
| コードスパゲッティ化 | どこに何があるか分からない |
| データ消失 | localStorage構造の衝突 |

## 振り返り（10分・チームで議論）
- **何が壊れた？** 具体的にどの機能がどう壊れた？
- **仕様の抜け** — 1人1票？取消？並び順？ AIは勝手に判断した？それとも無視した？
- **AIは何を知らなかった？** 既存コードの構造？データの保存方法？コンポーネントの依存関係？
- **どうすれば防げた？** → 事前に何を伝えておけば正しく動いた？ → **仕様を書こう。**

## ⏭ 次のPhaseへ
振り返りが終わったら、ファシリテーターの指示で `2-sdd/` に移動します。
Phase 2 で壊れたコードは置いていきます。Phase 3 ではクリーンな状態から再出発します。
