# 1-vibe-coding: Phase 1 & 2 — Vibe Coding

> **"ランチのお店を5件登録できて、ルーレットで1件選ぶWebアプリを作って。見た目もいい感じで。"**
> — この一言だけでアプリを作る。それが Vibe Coding。

## このリポジトリについて

Vibe Coding（雰囲気コーディング）を体験するためのリポジトリです。
仕様書なし、設計なし、思いつきのプロンプトだけでAIにアプリを作らせます。

Phase 1 で「速さ」に感動し、Phase 2 で「限界」を体感します。

### 2つの進め方

| 方法 | 説明 | おすすめ |
|---|---|---|
| **ゼロから挑戦** | プロンプトだけでアプリを作る — Vibe Codingの本質 | **推奨** |
| **セーフティネット** | 時間内に動かなかった場合、既存コードをベースにPhase 2へ進む | 予備 |

> `src/` 以下にはPhase 1の完成例（ルーレットアプリ）が入っています。まずはゼロから挑戦してみてください。

---

## セットアップ

```bash
# 1. このディレクトリに移動
cd repos/1-vibe-coding
```

## Phase 1（10分）: ランチルーレットアプリを爆速で作る

### 目的
とにかく"動くもの"を爆速で作る体験。

### やること
1. Claude Code が起動したら、`prompts/phase1-ルーレットアプリ.md` のプロンプトをコピペして入力
2. AIが生成したアプリを確認
3. 動いたら成功!

### プロンプト例（わざと雑め）
```
ランチのお店を5件登録できて、ルーレットで1件選ぶ
Webアプリを作って。見た目もいい感じで。
```

### 要件
- お店を5件登録できる
- ルーレットでランダムに1件選べる
- リロードしてもデータが保持される
- データ保存は localStorage（DB禁止）

---

## Phase 2（10分）: 投票機能を追加して「壊す」

### 目的
Phase 1 のアプリに機能追加して「壊れる」体験をする。

### ルール
- **Claude Code は閉じない** — Phase 1 と同じセッションで続ける
- **仕様書は書かない**（Vibe Codingを継続）
- 壊れてもOK — むしろ壊れることが学び

### やること
1. **Phase 1 と同じ Claude Code セッションのまま続ける**（新しく起動しない）
2. `prompts/phase2-投票機能追加.md` のプロンプトをコピペして入力
3. 何が起きるか観察する

### プロンプト例
```
チームで事前に候補を絞りたいから、投票機能つけて
```

### 起きるかもしれないこと

| 症状 | 具体例 |
|---|---|
| 状態管理破綻 | 投票数とUIが不一致 |
| 既存機能崩壊 | ルーレットが動かなくなる |
| コードスパゲッティ化 | どこに何があるか分からない |
| データ消失 | localStorage構造の衝突 |

---

## 振り返り（Phase 1 & 2 合同・10分）

チームで以下を議論してください:

- **速かった?** 最初に動いたのは何分後?
- **コード説明できる?** 生成されたコードの中身を理解している?
- **明日続きを書ける?** 引き継ぎドキュメントはある?
- **何が壊れた?** 仕様の抜け（1人1票? 取消? 並び順?）がどう影響した?
- **AIは何を知らなかった?** 既存コードの構造? データの保存方法?
- **どうすれば防げた?** → **仕様を書こう。**

## リポジトリ構成

```
1-vibe-coding/
├── CLAUDE.md                          # 最小限のプロジェクト説明
├── package.json                       # Next.js 14 + TypeScript + Tailwind CSS
├── prompts/
│   ├── phase1-ルーレットアプリ.md      # Phase 1 用プロンプト集
│   └── phase2-投票機能追加.md          # Phase 2 用プロンプト集
└── src/                               # Phase 1 完成例（セーフティネット）
    ├── app/
    │   ├── layout.tsx
    │   ├── page.tsx
    │   └── globals.css
    ├── components/
    │   ├── RestaurantCard.tsx          # レストランカード
    │   ├── RestaurantList.tsx          # レストラン一覧
    │   └── Roulette.tsx               # ルーレット
    ├── lib/
    │   ├── types.ts                   # 型定義
    │   └── storage.ts                 # localStorage ラッパー
    └── data/
        └── restaurants.ts             # 初期データ
```

| 観点 | 見るポイント | 計測方法 |
